1.5 GitHub Pages y Mkdocs

1.5.1 GitHub Pages

Creamos el repositorio donde vamos a alojar nuestra página web y en la configuración de ese repositorio, activamos Pages en la rama main usando la carpeta docs.

1.5.2 Mkdocs

Para usar mkdocs:

  1. Con mkdocs new miweb, creamos un nuevo proyecto mkdocs llamado 'miweb'.

  2. Entramos en el directorio miweb.

  3. Hacemos dentro de miweb un mkdocs build para crear los archivos de una página web.

  4. Con mkdocs serve, construye un sitio web y lo muestra en 127.0.0.1.

  5. Con mkdocs gh-deploy creamos una rama, publicamos los cambios en GitHub y te crea la página web pública.

  6. En el mkdocs.yml que se crea, podemos personalizar cosas como el tema, el nombre de la página, los plugins que añadimos... etc.

1.5.2.1 Instalar Mkdocs con venv de Python

  1. Instalamos Python seleccionando "Add Python.exe to PATH" si es en Windows y si es Linux, hacemos sudo apt install python3 y sudo apt install python3-pip.

  2. Probamos la instalación haciendo python --version y pip --version en la bash.

  3. Clonamos el repositorio que vayamos a usar para la página en nuestra bash.

  4. Entramos en el repositorio clonado.

  5. Hacemos python -m venv venv para crear un entorno virtual con el nombre de venv (virtual environment). Si en vez de Python tenemos python3, tenemos que instalar la opción de entorno virtual con sudo apt install python3-venv.

  6. Hacemos source venv/scripts/activate para activar el entorno virtual. Si en vez de python tenemos python3, tenemos que activarlo con source venv/bin/activate.

  7. Para desactivarlo, hacemos deactivate.

  8. Para borrar el entorno virtual, borramos el directorio donde creamos el entorno virtual.

  9. Para instalar la utilidad de mkdocs en nuestro entorno virtual, hacemos pip install mkdocs.

  10. Por último, para comprobar los paquetes pip de Python que hay instalados hacemos pip freeze.

1.5.2.2 Instalar Mkdocs con venv de Poetry

  1. Después de instalar Python, instalamos Poetry con sudo apt install python3-poetry.

  2. Con el comando poetry, podemos ver qué podemos hacer con él.

  3. Con poetry new nombre_proyecto, creamos un nuevo proyecto Python en nuestro directorio Home. En esta práctica, le hemos dado el nombre de publicar, creándolo con poetry new publicar.

  4. Entramos en el proyecto con cd publicar y con poetry install instalamos lo básico en nuestro proyecto Python.

  5. Para activar nuestro entorno virtual, hacemos poetry shell.

  6. Para salirnos de nuestro entorno virtual, hacemos exit.

  7. Ahora, vamos a instalar paquetes pip dentro de nuestro entorno virtual. Para instalar MkDocs, hacemos poetry add mkdocs.
