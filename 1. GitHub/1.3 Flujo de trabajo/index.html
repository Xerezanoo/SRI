1.3 Flujo de trabajo

El flujo de trabajo es simple:

  1. Trabajo con los archivos en mi PC y cuando termino, veo en qué ESTADO están (git status).

  2. Lo que está en rojo son los cambios que han habido y los que quiero AÑADIR a GitHub, los meto dentro de git add. Si quiero añadir TODO lo que hay, hago git add ..

  3. Para CONFIRMAR los cambios, hacemos un git commit y le añadiremos un mensaje para que estén más ordenados. Se puede poner lo que quieras, pero se suele poner "Refactor" o "Actualización" (git commit -m Actualización).

  4. Para SUBIR los cambios a GitHub, hacemos git push.

  5. Cuando vaya a otro ordenador, para VER si han habido cambios hago git fetch.

  6. Para TRAERME esos cambios a mi PC, hago git pull.

  7. Repito el flujo desde el paso 1.
